The DOM Series: The DOM Tree

by Ed Glazenski on March 19, 2016

When browsers create a model of an html document, they create a DOM tree. The DOM tree is stored into memory and it consists of four types of nodes: A document node, element nodes, attribute nodes, and text nodes. An illustration of a very basic DOM tree is shown below:

The document node is the gatekeeper of the document object. In order to access any of the nodes below the document node, you must start at the document node.

The teal colored nodes represent element nodes. In order to gain access to attribute and text nodes you must first select the desired element node.

Attribute nodes are not children of element nodes. They are part of their respective element node. Once you gain access to an element node, built-in javascript methods are provided to manipulate the attribute nodes within an element node.

Once you select an element node you have access to any text nodes that are contained within that element. Text nodes branch off of element nodes as their own nodes but they cannot have any children nodes branch off of them. When you have a text node this is the end of the line in regards to branching further off that node.

Every node is an object with methods and properties. These methods and properties allow you to target specific nodes or return NodeLists, which are collections of nodes. This makes for some pretty powerful possibilities.

In the next part for this series I will talk about different methods we can use to target specific elements and manipulate those elements along with attribute and text nodes. Thanks for visiting and have a great day!
